About Our Project

This project analyzes Geodata to map trends in hashtags on Twitter. By mapping these hastags, we can see the exact location of people who are tweeting about trending hashtags as well as what they are saying. By looking at these trends on the map we will correlate them to events that are happening in real life. For instance, you can see trends in world events going on by looking at tweets such as #NepalEarthquake or #Baltimore Riots. This app also can help you find things that are trending on the internet by looking at hastags such as #MattsNewVideo. Our map is helpful in order to see what parts of the world and country are concerned about what events.

Technologies Used

Cassandra

All of our tweet storage is done using Cassandra and the Datastax api. As a robust NoSQL database, it was perfect for storing large amounts of data that can be accessed via a tweet Id.



Django/Python3

We created our web frontend using the Django Python3 framework and using Apache to serve it. Since we had so many python experts in the group, it seemed reasonable to use a python based front-end for delivering our content.



Storm

We were able to do real time processing on twitter's Firehose API using the powerfull capabilities of Storm. This platform helped us parse tweets, count hashtags, store our data into our Cassandra Database, and all while new data was constantly streaming in.



Elastic Search

We implemented Elastic Search in order to be able to search for a desired hashtag in our database. On the App, the user can type a hashtag they want to see into the searchbar. After they search a hashtag, all other hashtags that appear in a tweet with the disired hashtag are displayed on the page

Google Maps API

We used the Google Maps API to bring in a map to plot our tweets. In our App, in conjunction with our Elastic Search implementation, you can search a hashtag, and then with the Google Maps API, it will plot all of the tweets in our database that contain that hashtag at the correct location on the map.



D3

We used D3 to implement a bar graph to display the top trending hashtags from all tweets in our database. We used a json object to store our top 200 hashtags, and then read them into a D3 bar chart. We connected our D3 bar chart to our map so that when you click on a hashtag in the bar chart, it will display those hashtags on the map.



AWS

Our Cassandra Database, Storm Cluster, and Django website are all hosted on an expandable Amazon Web Service Elastic Compute Cloud Instance.



Maven

Maven controlled our java dependencies making storm deploys easy and quick. Besides its extreme lack of documentation, maven can save a lot of heartache as a project progresses.
